# Textile Workshop ERP — О проекте

## Что это?

**Textile Workshop ERP** — это веб-приложение для управления швейным цехом. Система заменяет ручной учёт (тетради, Excel, устные договорённости) на единую цифровую платформу, где все данные о заказах, материалах, производстве и зарплатах находятся в одном месте.

**Адрес системы:** https://textile-workshop-erp.com

---

## Какую проблему решает?

Малые и средние швейные цеха сталкиваются с типичными проблемами:

| Проблема | Как система решает |
|----------|-------------------|
| Заказы теряются, статусы непонятны | Все заказы в таблице с фильтрами, статус виден сразу |
| Непонятно сколько ткани осталось на складе | Учёт материалов с уведомлениями о низком остатке |
| Ручной подсчёт сколько сшито за день | Ежедневные отчёты производства |
| Ошибки в расчёте зарплат (сдельная оплата) | Автоматический расчёт: кол-во изделий × ставка |
| Материалы списывают вручную и с ошибками | Автоматическое списание по техкарте продукта |
| Нет понимания прибыльности | Дашборд с выручкой, статистикой, графиками |

## Модули системы

### 1. Дашборд (Главная страница)
Обзор всего бизнеса на одном экране:
- Выручка за месяц
- Количество заказов (всего / активных / завершённых)
- Предупреждения о низком остатке материалов
- График производства за неделю
- Таблица последних заказов

### 2. Заказы
Полный цикл управления заказами:
- Создание заказа (клиент, модель, количество, дедлайн)
- Автоматический расчёт суммы заказа
- Отслеживание статуса: `Новый → В работе → Раскрой → Пошив → Упаковка → Отправлен → Завершён`
- Фильтрация и поиск
- Редактирование и удаление

### 3. Клиенты
База данных всех заказчиков:
- Имя / название компании, телефон, адрес
- Количество заказов от каждого клиента
- Поиск по имени или телефону

### 4. Склад (Инвентарь)
Учёт всех материалов — тканей, фурнитуры, ниток:
- Текущий остаток и минимальный порог
- Красная подсветка позиций с критическим остатком
- Регистрация закупок (количество + стоимость)
- При закупке остаток автоматически увеличивается

### 5. Техническая карта (Bill of Materials)
Для каждого продукта задаётся, сколько какого материала нужно на 1 единицу. Например:
- Пальто кашемир: 2.5м кашемира + 2м подкладки + 6 пуговиц + 1 молния
- Юбка хлопок: 1.5кг хлопка + 1 молния

Когда создаётся отчёт о производстве — материалы **автоматически списываются** со склада.

### 6. Производство
Ежедневные отчёты:
- Выбрать заказ и сотрудника
- Указать дату, количество сшитых / упакованных / бракованных изделий
- При сохранении:
  - Материалы списываются автоматически
  - Если материалов не хватает — система покажет ошибку
  - Данные попадают в расчёт зарплаты

### 7. Сотрудники и зарплаты
Управление персоналом и расчёт сдельной оплаты:
- Список сотрудников с должностью и ставкой за изделие
- Расчёт зарплаты за любой месяц
- **Формула:** `зарплата = (сшито − брак) × ставка за изделие`
- Итоговая сумма к выплате по всем сотрудникам

### 8. Отчёты
Графики и аналитика:
- Доходы и расходы (столбчатая диаграмма)
- Динамика производства (линейный график)
- Распределение заказов по статусам (круговая диаграмма)
- Топ продукция по выручке

## Технологии

| Компонент | Технология |
|-----------|-----------|
| Интерфейс (frontend) | React.js + Tailwind CSS |
| Сервер (backend) | Java Spring Boot |
| База данных | PostgreSQL |
| Хранение файлов | MinIO (S3-совместимый) |
| Авторизация | JWT токены |
| Языки интерфейса | Русский, English, Кыргызча |
| Развёртывание | Docker + Nginx + SSL (Let's Encrypt) |

## Архитектура

```
Пользователь (браузер)
    ↓
Nginx (SSL, порты 80/443)
    ├── /api/*  → Spring Boot (бизнес-логика, порт 8080)
    │                ├── PostgreSQL (данные)
    │                └── MinIO (файлы/изображения)
    └── /*      → React (интерфейс)
```

## Ключевые автоматизации

1. **Расчёт суммы заказа:** цена продукта × количество
2. **Списание материалов:** при создании отчёта производства материалы вычитаются со склада по техкарте
3. **Блокировка при нехватке:** если на складе недостаточно материала — отчёт не создаётся
4. **Расчёт зарплат:** автоматически из отчётов производства за выбранный период
5. **Пополнение склада:** при регистрации закупки остаток увеличивается

---

## Для кого предназначена система

- **Владелец / менеджер цеха** — видит всю картину: заказы, финансы, склад
- **Производственный персонал** — вводит ежедневные отчёты
- **Складской работник** — следит за остатками, регистрирует закупки
- **Бухгалтер** — использует расчёт зарплат и финансовые отчёты
